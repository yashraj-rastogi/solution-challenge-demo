Project Draft
Project Name: Crisis OS
Theme: Rapid Crisis Response
Domain: Hospitality crisis coordination for hotels and hostels (hospital workflow marked as future scope)

1. Problem We Are Solving
Hospitality venues face emergencies like fire, gas leaks, food poisoning, power outages, and security incidents. During these events, communication is fragmented across calls, WhatsApp, and manual updates.

Current pain points:
- Guests receive delayed or unclear instructions.
- Managers do not have one live command view.
- Staff updates are difficult to aggregate by room/floor.
- Responder handoff data is often incomplete.

This leads to confusion, panic, delayed interventions, and reputational damage.

2. Updated Solution
Crisis OS is a single web app with role-based dashboards that synchronizes communication across all participants in a crisis.

Unified workflow:
Report -> Analyze -> Broadcast -> Track -> Respond

The product combines:
- Hospitality Guard: instant alerting
- RespondeSync: live command dashboard
- SafetyAgents: AI-assisted next actions and escalation guidance

3. Roles and Dashboard Model
Roles:
- Org Admin
- Manager
- Staff
- Guest
- Responder (read-only)

Dashboard strategy:
- Admin setup dashboard for organization onboarding
- Manager command dashboard for live incident operations
- Staff mobile task dashboard for reporting and checklists
- Guest safety interface for alerts and status actions
- Responder read-only handoff screen

4. Device Strategy
- Guest and Staff: mobile-first web
- Manager: desktop-first with mobile-safe controls
- Org Admin: desktop-first
- Responder: mobile and desktop read-only

5. Organization Onboarding Flow
1. Org admin creates organization and property.
2. Org admin sets floors, rooms, and zones.
3. Org admin uploads map image for floor guidance.
4. Org admin adds manager and staff accounts.
5. Org admin enables guest join methods:
   - property code
   - room QR
6. Front desk asks guests to join at check-in.
7. Org admin runs a drill test and activates property.

6. Crisis Response Flow
1. Staff/manager creates incident draft.
2. Gemini generates structured summary, severity, and instructions.
3. Manager reviews and broadcasts targeted alert.
4. Guests submit status:
   - Safe
   - Need Help
   - Unable to Move
5. Dashboard updates by room/floor/zone in real time.
6. Staff executes checklists and posts updates.
7. Manager shares responder handoff summary.
8. Manager resolves incident and publishes all-clear.

7. MVP Recommendation 1: MVP A (Fast and Reliable)
Must-have features:
- Role-based login and route guards
- Admin setup basics (org/property/layout/guest access)
- Property code and room QR guest join flow
- Incident creation and AI draft generation
- Broadcast workflow
- Guest safety check-in
- Live response counters
- Responder handoff summary
- Incident resolve and all-clear

Why this is best now:
- Lowest risk for 3-day timeline
- Clear end-to-end demo
- Strong AI narrative

8. MVP Recommendation 2: MVP B (Impact Upgrade)
Add only after MVP A is stable:
- Floor/zone map guidance overlay
- Voice-to-text staff reporting
- Expanded multilingual support
- AI escalation recommendation panel
- Mock sensor-triggered incident inputs
- PWA/offline behavior

9. Recommended Stack
Primary:
- React + Vite
- Tailwind CSS
- Firebase
- Gemini API
- Firebase Hosting or Vercel

Optional:
- Firebase Cloud Messaging
- Google Maps API or image-based map overlay for demo
- Browser speech-to-text

Teammate 2 (Backend + AI):

Shared:

Day 2:

Day 3:

- Record demo and finalize pitch

12. Manual Steps Included
- Front desk onboarding of guests via code/QR
- Shift drill check by manager
- Manual PA fallback if broadcast fails
- All-clear checklist before closure

13. What We Are Not Building in MVP
- Real IoT integrations
- Hardware speaker integrations
- Full autonomous multi-agent orchestration
- Enterprise analytics suite
- Full hospital workflow implementation

14. One-Line Pitch
Crisis OS turns fragmented hospitality emergency communication into real-time, multilingual, role-based coordinated action.
